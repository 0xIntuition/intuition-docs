# Contracts

This page provides a comprehensive list of all Intuition smart contracts with their deployment addresses, ABIs, and key information.

## Core Contracts

### **EthMultiVault.sol**

The main vault contract that handles deposits, redemptions, and share management for the Intuition protocol.

| Feature | Description |
|---------|-------------|
| **Purpose** | Multi-vault management for ETH deposits and redemptions |
| **Key Functions** | Deposit, redeem, batch operations, curve management |
| **Security** | Audited by Consensys Diligence |
| **Upgradeable** | Yes (via proxy pattern) |

<details>
<summary>**Contract Details**</summary>

#### **Key Functions**

```solidity
// Core deposit and redemption
function deposit(address receiver, uint256 termId) external payable returns (uint256);
function redeem(uint256 percentage, address receiver, uint256 termId) external returns (uint256);

// Batch operations
function batchDeposit(address receiver, uint256[] calldata termIds, uint256[] calldata amounts) external payable returns (uint256[] memory);
function batchRedeem(uint256 percentage, address receiver, uint256[] calldata ids) external returns (uint256[] memory);

// Bonding curve operations
function depositAtomCurve(address receiver, uint256 atomId, uint256 curveId) external payable returns (uint256);
function redeemAtomCurve(uint256 percentage, address receiver, uint256 atomId, uint256 curveId) external returns (uint256);

// View functions
function currentSharePriceCurve(uint256 vaultId, uint256 curveId) external view returns (uint256);
function maxRedeemCurve(address owner, uint256 vaultId, uint256 curveId) external view returns (uint256);
```

#### **Events**

```solidity
event Deposit(address indexed receiver, uint256 indexed termId, uint256 assets, uint256 shares);
event Redeem(address indexed receiver, uint256 indexed termId, uint256 assets, uint256 shares);
event BatchDeposit(address indexed receiver, uint256[] termIds, uint256[] assets, uint256[] shares);
event BatchRedeem(address indexed receiver, uint256[] termIds, uint256[] assets, uint256[] shares);
```

</details>

### **AtomWallet.sol**

Smart contract wallet implementation for managing atom-specific operations and permissions.

| Feature | Description |
|---------|-------------|
| **Purpose** | Smart contract wallet for atom management |
| **Key Functions** | Atom creation, triple creation, signal management |
| **Integration** | Works with EthMultiVault for economic operations |
| **Security** | Role-based access control |

<details>
<summary>**Contract Details**</summary>

#### **Key Functions**

```solidity
// Atom operations
function createAtom(bytes calldata atomUri) external payable returns (uint256);
function batchCreateAtom(bytes[] calldata atomUris) external payable returns (uint256[] memory);

// Triple operations
function createTriple(uint256 subjectId, uint256 predicateId, uint256 objectId) external payable returns (uint256);
function batchCreateTriple(uint256[] calldata subjectIds, uint256[] calldata predicateIds, uint256[] calldata objectIds) external payable returns (uint256[] memory);

// Signal management
function signal(uint256 termId, uint256 amount) external payable;
function batchSignal(uint256[] calldata termIds, uint256[] calldata amounts) external payable;

// View functions
function getAtomCost() external view returns (uint256);
function getTripleCost() external view returns (uint256);
```

#### **Events**

```solidity
event AtomCreated(uint256 indexed atomId, address indexed creator, bytes atomUri);
event TripleCreated(uint256 indexed tripleId, uint256 indexed subjectId, uint256 indexed predicateId, uint256 objectId);

```

</details>

### **Errors.sol**

Custom error definitions for the Intuition protocol, providing clear and gas-efficient error handling.

| Feature | Description |
|---------|-------------|
| **Purpose** | Centralized error definitions |
| **Benefits** | Gas efficient, clear error messages |
| **Usage** | Used across all Intuition contracts |
| **Maintenance** | Single source of truth for errors |

<details>
<summary>**Contract Details**</summary>

#### **Error Definitions**

```solidity
// Vault errors
error Vault__InsufficientShares(uint256 requested, uint256 available);
error Vault__InsufficientAssets(uint256 requested, uint256 available);
error Vault__InvalidCurveId(uint256 curveId);
error Vault__InvalidTermId(uint256 termId);

// Atom errors
error Atom__InvalidUri(bytes uri);
error Atom__AlreadyExists(uint256 atomId);
error Atom__DoesNotExist(uint256 atomId);
error Atom__InsufficientPayment(uint256 required, uint256 provided);

// Triple errors
error Triple__InvalidSubject(uint256 subjectId);
error Triple__InvalidPredicate(uint256 predicateId);
error Triple__InvalidObject(uint256 objectId);
error Triple__AlreadyExists(uint256 tripleId);
error Triple__DoesNotExist(uint256 tripleId);

// Signal errors
error Signal__InvalidAmount(uint256 amount);
error Signal__InsufficientBalance(uint256 requested, uint256 available);
error Signal__InvalidTerm(uint256 termId);

// Batch operation errors
error Batch__ArrayLengthMismatch(uint256 array1Length, uint256 array2Length);
error Batch__EmptyArray();
error Batch__InvalidPercentage(uint256 percentage);

// Approval errors
error Approval__InvalidApprovalType(uint256 approvalType);
error Approval__Unauthorized(address sender, address owner);
error Approval__InvalidSender(address sender);
```

#### **Usage Examples**

```solidity
// In other contracts
if (shares < requestedShares) {
    revert Vault__InsufficientShares(requestedShares, shares);
}

if (msg.value < atomCost) {
    revert Atom__InsufficientPayment(atomCost, msg.value);
}

if (subjectId == 0 || predicateId == 0 || objectId == 0) {
    revert Triple__InvalidSubject(subjectId);
}
```

</details>

## Integration Examples

### **Web3.js Integration**

```javascript
import Web3 from 'web3';
import EthMultiVaultABI from './abis/EthMultiVault.json';

const web3 = new Web3(window.ethereum);
const vaultContract = new web3.eth.Contract(
  EthMultiVaultABI,
  '0x...' // Contract address
);

// Deposit ETH
async function deposit(receiver, termId, amount) {
  try {
    const result = await vaultContract.methods
      .deposit(receiver, termId)
      .send({ from: receiver, value: amount });
    return result;
  } catch (error) {
    console.error('Deposit failed:', error);
  }
}
```

### **React Integration**

```jsx
import { useContract, useProvider, useSigner } from 'wagmi';
import { EthMultiVaultABI } from './abis';

function VaultInterface() {
  const provider = useProvider();
  const { data: signer } = useSigner();
  
  const vaultContract = useContract({
    address: '0x...', // Contract address
    abi: EthMultiVaultABI,
    signerOrProvider: signer || provider,
  });

  const handleDeposit = async (termId, amount) => {
    try {
      const tx = await vaultContract.deposit(signer.getAddress(), termId, {
        value: amount
      });
      await tx.wait();
      console.log('Deposit successful!');
    } catch (error) {
      console.error('Deposit failed:', error);
    }
  };

  return (
    <div>
      <button onClick={() => handleDeposit(1, ethers.utils.parseEther('0.1'))}>
        Deposit 0.1 ETH
      </button>
    </div>
  );
}
```
