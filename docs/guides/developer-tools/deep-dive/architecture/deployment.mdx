## Network Layer Deployment

### Validator Nodes

```yaml
# Kubernetes deployment for validator nodes
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: intuition-validator
spec:
  serviceName: intuition-validator
  replicas: 5
  selector:
    matchLabels:
      app: intuition-validator
  template:
    metadata:
      labels:
        app: intuition-validator
    spec:
      containers:
      - name: validator
        image: intuition/validator:latest
        resources:
          requests:
            memory: "8Gi"
            cpu: "4"
          limits:
            memory: "16Gi"
            cpu: "8"
        volumeMounts:
        - name: data
          mountPath: /data
  volumeClaimTemplates:
  - metadata:
      name: data
    spec:
      accessModes: ["ReadWriteOnce"]
      resources:
        requests:
          storage: 1Ti
```

### RPC Nodes

Multi-region deployment for low latency:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: intuition-rpc
spec:
  replicas: 10
  selector:
    matchLabels:
      app: intuition-rpc
  template:
    metadata:
      labels:
        app: intuition-rpc
    spec:
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            podAffinityTerm:
              topologyKey: topology.kubernetes.io/zone
      containers:
      - name: rpc
        image: intuition/rpc:latest
        resources:
          requests:
            memory: "4Gi"
            cpu: "2"
          limits:
            memory: "8Gi"
            cpu: "4"
        readinessProbe:
          httpGet:
            path: /health
            port: 8545
          initialDelaySeconds: 30
          periodSeconds: 10
```

### Rolling Updates

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: event-consumer
spec:
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  # ...
```

### Blue-Green Deployments

```bash
# Deploy new version (green)
kubectl apply -f deployment-green.yaml

# Wait for health checks
kubectl wait --for=condition=available deployment/app-green

# Switch traffic
kubectl patch service app -p '{"spec":{"selector":{"version":"green"}}}'

# Monitor and rollback if needed
kubectl patch service app -p '{"spec":{"selector":{"version":"blue"}}}'
```

### Canary Releases

```yaml

kind: Canary
metadata:
```

## Disaster Recovery

### Backup Strategy

```bash
# PostgreSQL backups
# - Continuous WAL archiving
# - Daily full backups
# - 30-day retention

# Backup script
#!/bin/bash
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
pg_dump -h $DB_HOST -U $DB_USER $DB_NAME | \
  gzip > /backups/intuition_${TIMESTAMP}.sql.gz

# Upload to S3
aws s3 cp /backups/intuition_${TIMESTAMP}.sql.gz \
  s3://intuition-backups/postgres/
```

### Recovery Procedures

1. **Network Recovery**
   - Restore from latest snapshot
   - Replay blocks from checkpoint
   - Validate state consistency

2. **Database Recovery**
   - Restore from backup
   - Apply WAL archives
   - Verify data integrity

3. **Service Recovery**
   - Redeploy from last known good version
   - Restore configurations
   - Verify connectivity

### Horizontal Scaling

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: event-consumer-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: event-consumer
  minReplicas: 3
  maxReplicas: 20
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Pods
    pods:
      metric:
        name: sqs_queue_depth
      target:
        type: AverageValue
        averageValue: "1000"
```

### Network Policies

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: intuition-network-policy
spec:
  podSelector:
    matchLabels:

  policyTypes:
```
